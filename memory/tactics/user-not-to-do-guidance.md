# User Not-To-Do Guidance

## Purpose
Define how Plenra communicates risky actions the user should avoid in a given decision context.

## Guidance Principles
- guidance must be tied to the actual decision context
- guidance must be specific
- guidance must reduce harmful action
- guidance must not increase confusion
- guidance must not be generic fear-based messaging

## Example Output Types
- do not stop the campaign immediately
- do not raise budget under panic
- do not change multiple variables at once
- do not conclude from a single day of data
- do not override a HOLD decision without review

## Trigger Contexts
- HOLD
- STOP
- COOLDOWN
- unstable signals
- high-risk decisions
- incomplete data environment
